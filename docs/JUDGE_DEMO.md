# CrimeSight: 3-minute Judge Demo

## 1. Start with the problem — 15 seconds

“Crime data is hard to query quickly, and a generic AI answer cannot be trusted without evidence. CrimeSight turns a question into a verified FIR result before a human takes action.”

## 2. Show Query Copilot — 40 seconds

Open the bottom **Ask CrimeSight AI** panel and select:

> Show high-risk cybercrime FIRs in Mysuru.

Point out the count, filters, matching FIR records, risk scores, query ID, and synthetic-data boundary.

Say: “This is not LLM-generated SQL. CrimeSight uses allow-listed filters and exposes the records that support the answer.”

## 3. Use Demo Mode — 70 seconds

Click **Demo Mode** in the header.

1. **Query Proof:** verified query, visible filters, matching FIRs.
2. **Challenge:** click “Challenge this interpretation” for “Show serious cases.” Explain that CrimeSight refuses to guess what serious means.
3. **Connected case:** show FIR, linked context, and evidence readiness.
4. **Human decision:** approve review or request evidence. Explain that this records a decision; it does not automate enforcement.
5. **Field to action:** open the Field FIR flow and explain the Catalyst persistence path.

## 4. Close with the differentiator — 20 seconds

“CrimeSight is not an AI that tells police what to do. It gives officers verified intelligence, lets them challenge the interpretation, and records the human decision before action.”

## Judge Challenge Pack

Invite judges to try these:

1. `How many repeat-pattern FIRs are open?`
2. `What is the most common crime in Bengaluru Urban?`
3. `Show theft FIRs in Ballari from the last 30 days`
4. `Show serious cases` — should request clarification.
5. `Show cybercrime FIRs in Kodagu` — may safely return no matching records.
